# Project Mandates (AreaHustle)

## Operational Mandates
- **Error Logging:** If an implementation attempt fails, you MUST document the failure, including the error details and technical context, in a dedicated `FAILED_ATTEMPTS.md` file before attempting an alternative approach.
- **Surgical Edits:** Prioritize targeted, surgical edits over large-scale refactors to maintain stability and minimize code noise.
- **Research First:** Use the `web_search` and `web_fetch` tools extensively for technical queries. Do NOT over-rely on internal knowledge or assume API availability without verification.
- **No Guessing:** If requirements are ambiguous or technical confusion arises, you MUST stop and ask the user for clarification. Never guess at intended functionality or API behavior.
- **Index Maintenance:** Whenever a new documentation file (`.md`) is created or an existing one is significantly renamed, you MUST update `INDEX.md` to reflect the change.
